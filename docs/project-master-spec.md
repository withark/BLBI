# 프로젝트 통합 설계 문서 (Project Master Spec)

## 1. 서비스 목표
- 서비스명: 블비
- 핵심 대상: 지역 음식점/카페 소상공인 사장님
- 핵심 문제: 블로그 운영 시간을 줄이고, 키워드 기반으로 검색 친화 글을 빠르게 확보
- 핵심 방식: 키워드 1개 입력 중심의 자동 운영 도구

## 2. 사용자/UX 원칙
- 사용자 특성: 컴퓨터/웹에 익숙하지 않을 수 있음
- UX 원칙: 쉽고 직관적이며 단순해야 함
- 고정 원칙:
  - 큰 입력창 1개 + 생성 버튼 1개
  - 채팅형 UX 금지
  - 추천 칩/예시는 보조로만 배치 (입력창보다 앞 금지)
  - 메인은 도구처럼, 추천은 보조처럼

## 3. 기능 기준 (MVP 우선)
- 가게 정보 등록/수정 (BusinessProfile)
- 키워드 1개로 블로그 생성
- 생성 결과 저장/히스토리 조회
- 다음 글 추천
- 시리즈 주제 생성 (Premium)
- 네이버 블로그 복붙 최적화 결과 제공
- 플랜/한도 적용 + 업그레이드 안내
- 관리자 기본 지표

## 4. 출력/복사 규칙 (절대 기준)
- 복사 결과는 순수 텍스트(export_text)
- HTML/Markdown/코드블록 금지
- 구조: 소제목 → 본문 → 사진 가이드
- 사진 가이드 포맷: "사진 가이드 : 한 줄 설명"
- 화면 표시 결과와 복사 결과가 최대한 일치해야 함

## 5. 요금제 정책
- Free: 일 3회
- Basic: 월 15회
- Premium: 월 무제한
- 시리즈 주제 생성은 Premium 전용
- 한도 계산은 단일 함수/단일 소스에서 처리

## 6. 연결성/운영 원칙
- Dead button/link/page 금지
- 동작 불가 기능은 명확한 "준비 중" 안내 제공
- 프론트-API-저장-관리자 연결이 실제로 이어져야 함

## 7. 구현 원칙
- 기존 코드 구조/컴포넌트/라우팅을 참고하지 않음
- 본 문서와 기획 문서(UX/PRD/GENERATION_RULES 등)를 기준으로 신규 설계
- 기존 코드의 한계를 가져오지 않고 목표 달성에 유리한 구조로 구현
